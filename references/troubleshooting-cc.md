# CC Flink SQL — Troubleshooting

CC-specific errors with causes and fixes. Excludes OSS-only errors.

## SQL Parse Errors

### "Encountered \<reserved-word\>"
```
SQL parse failed. Encountered "timestamp" at line N, column M.
Was expecting one of: <BACK_QUOTED_IDENTIFIER> ...
```
**Cause:** Unquoted reserved word used as identifier.
**Fix:** Backquote the identifier — including inside ROW type aliases:
```sql
ROW<`timestamp` STRING, `value` STRING>
```

### "Unsupported format: json"
```
Unsupported format: json
```
**Cause:** Plain format names (`json`, `avro`, `csv`) not supported on CC.
**Fix:** Use SR-backed formats: `json-registry`, `avro-registry`, `proto-registry`, or `raw`.

### "LATERAL TABLE(UNNEST(...))" parse error
**Cause:** CC rejects `LATERAL TABLE(UNNEST(...))` syntax.
**Fix:** Use `CROSS JOIN UNNEST(...)`:
```sql
-- ❌ FAILS
FROM input LATERAL TABLE(UNNEST(arr)) AS T(elem)
-- ✅ WORKS
FROM input CROSS JOIN UNNEST(arr) AS T(elem)
```

## Function Errors

### "Function 'PROCTIME' is not supported"
```
Function 'PROCTIME' is not supported in Confluent's Flink SQL dialect.
```
**Cause:** CC has no `PROCTIME()`. Lookup join pattern doesn't compile.
**Fix (pick one):**
1. External Tables + `KEY_SEARCH_AGG` (canonical for DB/REST lookups)
2. Regular join against upsert-kafka topic (reference data in Kafka)
3. Event-time temporal join: `FOR SYSTEM_TIME AS OF o.order_time`

See `sql-patterns-cc.md` for full examples.

### "Non-deterministic function in update query"
```
The column(s): enriched_at (generated by non-deterministic function:
CURRENT_TIMESTAMP) can not satisfy the determinism requirement
```
**Cause:** Query produces updates (e.g., upsert join) and includes `CURRENT_TIMESTAMP`, `NOW()`, `RAND()`, `UUID()`.
**Fix:** Remove non-deterministic column, or carry timestamp from source row, or make query insert-only (temporal join instead of regular join).

## Sink Errors

### "Table sink doesn't support consuming update and delete changes"
```
Table sink '...' doesn't support consuming update and delete changes
```
**Cause:** Regular JOIN against upsert table emits updates, but sink is `changelog.mode = 'append'`.
**Fix:** Make sink upsert-capable:
```sql
CREATE TABLE enriched (
  order_id STRING NOT NULL,
  ...,
  PRIMARY KEY (order_id) NOT ENFORCED
) WITH ('changelog.mode' = 'upsert', ...);
```

## Statement Lifecycle Errors

### Statement stuck in PENDING > 60s
**Cause:** Resource scheduling, SQL error not yet surfaced, or carry-over waiting for v1 stop.
**Fix:**
```bash
confluent flink statement exception list <name> --cloud <provider> --region <region>
```
For carry-over: ensure v1 is STOPPED (not just submitted for stop).

### "expired token"
**Cause:** CLI OAuth token expired. Subprocess caches can go stale independently.
**Fix:**
```bash
confluent login --save
```

### Wrong flags on statement commands
```
Error: unknown flag: --cloud
```
**Cause:** `--cloud`/`--region` used on `statement create` (which only accepts `--environment`).
**Fix:** See CLI flag schema table in `cli-reference.md`.

## Schema Errors

### Table not found
```
Table 'topic_name' not found
```
**Cause:** Topic doesn't exist, wrong catalog path, or no schema in SR.
**Fix:**
```sql
SHOW CATALOGS;
SHOW DATABASES;
SHOW TABLES;
-- Use fully qualified: `cluster_id`.`database_name`.`topic_name`
```

### Schema inference mismatch
**Cause:** Flink-inferred schema from SR doesn't match expected types.
**Fix:**
```sql
DESCRIBE my_table;
```
Compare with producer schema. Register explicit schema if auto-inference is wrong.

## UDF Errors

### "Artifact not found"
**Cause:** Wrong artifact ID, artifact in different region, or artifact deleted.
**Fix:**
```bash
confluent flink artifact list
# Verify artifact ID matches USING JAR 'confluent-artifact://<id>'
```

### UDF egress denied
**Cause:** UDF tries network access without proper connection.
**Fix:** Create CONNECTION and declare `USING CONNECTIONS`:
```sql
CREATE CONNECTION my_conn WITH (...);
CREATE FUNCTION my_func AS 'com.example.MyFunc'
  USING JAR 'confluent-artifact://cfa-xxxxx'
  USING CONNECTIONS ('my_conn');
```

## CFU / Resource Errors

### "CFU limit exceeded"
```
Statement cannot be scheduled: compute pool CFU limit exceeded
```
**Fix:**
```bash
confluent flink compute-pool describe $POOL_ID
confluent flink statement list --cloud <provider> --region <region>
# Stop unused statements or increase max-cfu
```

## Consume Errors

### UTF-8 decode errors on topic consume
**Cause:** Default `--value-format string` on SR-backed topic with `id-encoding=confluent` — first 5 bytes are wire-format prefix.
**Fix:** Use `--value-format jsonschema` (or `avro`/`protobuf`).

### Interleaved `% Headers:` lines in output
**Cause:** `id-encoding=header` stores schema ID in Kafka headers. Consumer prints header debug lines.
**Fix:** Filter with `grep -v '^%'` when piping output.

## EXPLAIN for pre-validation

Before `statement create`, validate syntax with EXPLAIN:
```sql
EXPLAIN SELECT * FROM my_complex_query;
EXPLAIN PLAN FOR INSERT INTO output SELECT * FROM input;
```

Catches parse errors, type mismatches, and ambiguous columns without consuming CFUs. Add to verification loop as step 0.
